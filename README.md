# n8n-claw

A self-hosted AI Telegram agent built on n8n + Claude (Anthropic) + Supabase.
Send a message to your Telegram bot and Claude handles it — with memory, tool use,
MCP server support, reminders, and a workflow builder.

This repository contains all fixes baked in from the original
[n8n-claw](https://github.com/freddy-schuetz/n8n-claw) deployment.

---

## Architecture

```
Telegram  -->  n8n webhook  -->  n8n-claw Agent (Claude)
                                   |
                    +--------------+--------------+
                    |              |              |
               Supabase DB    MCP servers    Tool workflows
               (memory,       (weather, ...)  (reminders,
                soul, agents)               workflow builder,
                                            MCP builder)
```

**Five n8n workflows:**

| Workflow | Purpose |
|---|---|
| n8n-claw Agent | Main entrypoint — receives Telegram messages, calls Claude |
| MCP Client | Proxy to call any MCP server tool |
| MCP Builder | Builds new MCP servers on demand |
| Reminder Factory | Creates scheduled reminder workflows |
| Workflow Builder | Builds arbitrary n8n workflows on demand |

**Infrastructure:**

- **n8n** — workflow automation (native systemd service or Docker)
- **Supabase (self-hosted)** — PostgreSQL + PostgREST + Kong API gateway + Studio
- **Claude** — Anthropic API (Sonnet by default)

---

## Prerequisites

- A Linux server (Ubuntu 22.04/24.04 or Debian 12 recommended), root or sudo access
- A domain with HTTPS configured (nginx/Caddy reverse proxy pointing to port 5678)
- A Telegram bot token from [@BotFather](https://t.me/BotFather)
- An [Anthropic API key](https://console.anthropic.com/)
- Python 3 installed on the server (`python3`)
- `sqlite3` CLI installed (`apt install sqlite3`)
- Docker CE (setup.sh installs this automatically if missing)

## Quick Start

```bash
# 1. Clone the repo
git clone https://github.com/diogonmoura/n8n-claw.git /opt/n8n-claw
cd /opt/n8n-claw

# 2. Create and edit .env
cp .env.example .env
nano .env            # fill in N8N_URL, ANTHROPIC_API_KEY, TELEGRAM_BOT_TOKEN,
                     # TELEGRAM_CHAT_ID, SUPABASE_URL (your server LAN IP:8000)

# 3. Run setup (native n8n via systemd — recommended)
sudo bash setup.sh --native

# 4. Follow the printed instructions to create credentials in n8n UI,
#    then run the two setup scripts

# 5. Toggle the n8n-claw Agent workflow OFF then ON in n8n UI
```

After step 5, send your Telegram bot a message. It should respond.

## Detailed Setup

### Phase 1 — Automated (setup.sh)

`setup.sh` does the following:

1. Validates `.env` and generates missing Supabase secrets
2. Installs Docker CE (if not present)
3. For `--native` mode: installs Node.js 20 + n8n globally, sets up systemd service
4. Generates `supabase/kong.deployed.yml` (Kong config with real keys)
5. Starts the Supabase stack (PostgreSQL, PostgREST, Kong, Studio, pg-meta)
6. Runs database migrations (`001_schema.sql` + `002_seed_en.sql`)
7. Starts n8n

### Phase 2 — Manual: Create credentials in n8n UI

Open your n8n instance (`N8N_URL`) and complete first-run setup (admin user).

Then go to **Settings → Credentials** and create four credentials:

| Type | Name | Key fields |
|---|---|---|
| Telegram API | `n8n-claw Bot` | Access Token (from BotFather) |
| PostgreSQL | `Postgres (Supabase)` | host: 127.0.0.1, port: 5432, db: postgres, user: postgres, password from `.env` |
| Anthropic API | `Anthropic account` | API Key from Anthropic Console |
| HTTP Header Auth | `Kong (Supabase)` | Header Name: `apikey`, Header Value: `SUPABASE_ANON_KEY` from `.env` |

**The credential names must match exactly** (case-sensitive) — the wiring scripts use the `type` field to match.

### Phase 3 — Wire credentials and import workflows

```bash
# Import workflows + wire credentials + fix cross-references + activate
python3 scripts/setup-db.py --env-file .env

# After setup-db.py, run phase2 to confirm all credential IDs are wired
python3 scripts/phase2-wire-credentials.py --env-file .env
```

### Phase 4 — Sync Telegram webhook

In n8n UI, find the **n8n-claw Agent** workflow and toggle it **OFF**, then **ON**.
This re-registers the Telegram webhook with the secret token stored in n8n.

## Docker vs Native n8n

### Native (default, recommended)

n8n runs as a systemd service (`/etc/systemd/system/n8n.service`).
Config is at `/opt/n8n.env`. Data is in `/.n8n/`.

```bash
sudo bash setup.sh --native
systemctl status n8n
journalctl -u n8n -f
```

### Docker

n8n runs in a container alongside Supabase.
Data is in the `n8n_data` Docker volume.

```bash
sudo bash setup.sh --docker
docker logs n8n-claw-n8n -f
```

Note: with `--docker`, the `--n8n-folder` for the setup scripts should point to the
volume mount inside the container (typically `/home/node/.n8n`).

## Troubleshooting

**Bot does not respond:**
- Check n8n is running: `systemctl status n8n` or `docker ps`
- Check the n8n-claw Agent workflow is active in n8n UI
- Toggle the workflow OFF then ON to re-sync the Telegram webhook
- Check n8n logs: `journalctl -u n8n -f` or `docker logs n8n-claw-n8n -f`

**Credential errors in workflow executions:**
- Run `python3 scripts/phase2-wire-credentials.py --env-file .env`
- Verify credential names in n8n UI match exactly what the scripts expect

**PostgREST returns 401/403:**
- Check `SUPABASE_ANON_KEY` in `.env` matches the key in `supabase/kong.deployed.yml`
- Restart Kong: `docker restart n8n-claw-kong`

**Supabase containers not starting:**
- `docker compose -f docker-compose.supabase.yml logs`
- Ensure `supabase/kong.deployed.yml` exists (run `generate_kong` in setup.sh)
