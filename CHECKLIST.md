# n8n-claw Setup Checklist

**The only file you need to edit is `.env`.**
Everything else (secrets, workflow imports, credential wiring, Kong config, DB migrations) is handled automatically by `setup.sh` and the helper scripts.

---

## Step 1 — Fill in `.env`

Copy the example and open it:
```bash
cp .env.example .env
nano .env   # or vim, code, etc.
```

### You MUST fill in these (no defaults):

| Variable | Where to get it |
|---|---|
| `N8N_URL` | Your public n8n URL, e.g. `https://n8n.yourdomain.com` |
| `SUPABASE_URL` | `http://YOUR_SERVER_LAN_IP:8000` (LAN IP, not localhost) |
| `ANTHROPIC_API_KEY` | https://console.anthropic.com → API Keys |
| `TELEGRAM_BOT_TOKEN` | BotFather on Telegram → `/newbot` |
| `TELEGRAM_CHAT_ID` | Your personal Telegram user ID (from @userinfobot) |

### Fill in your profile (used to seed the DB):

| Variable | Example |
|---|---|
| `USER_TELEGRAM_ID` | `telegram:123456789` |
| `USER_NAME` | `john` |
| `USER_DISPLAY_NAME` | `John Doe` |
| `USER_TIMEZONE` | `Europe/Lisbon` |
| `USER_CONTEXT` | A short description of yourself and your preferences |

### Leave these EMPTY — auto-generated by `setup.sh`:

| Variable | What it is |
|---|---|
| `POSTGRES_PASSWORD` | Random password for PostgreSQL |
| `SUPABASE_ADMIN_PASSWORD` | Random password for the `supabase_admin` DB role |
| `SUPABASE_JWT_SECRET` | Random HS256 secret for Supabase JWTs |
| `SUPABASE_ANON_KEY` | Supabase anonymous JWT (auto-derived from JWT_SECRET) |
| `SUPABASE_SERVICE_KEY` | Supabase service role JWT (auto-derived) |

### Optional:

| Variable | Default | Notes |
|---|---|---|
| `BRAVE_API_KEY` | `no-brave-key` | Only needed if you want web search in MCP Builder |
| `N8N_PORT` | `5678` | Change only if port 5678 is taken |

---

## Step 2 — Run setup.sh

```bash
# Native install (n8n as systemd service) — recommended
sudo bash setup.sh --native

# Docker install (n8n in Docker)
sudo bash setup.sh --docker
```

`setup.sh` will:
- ✅ Detect OS (Debian / Ubuntu)
- ✅ Install Docker CE
- ✅ Install Node.js 20 + n8n (native mode)
- ✅ Generate all missing secrets and write them back to `.env`
- ✅ Generate `supabase/kong.deployed.yml` (fills in Supabase keys)
- ✅ Start Supabase stack (PostgreSQL, PostgREST, Kong, Studio)
- ✅ Run DB migrations (`001_schema.sql` with admin password substituted)
- ✅ Seed DB with your profile data from `.env`
- ✅ Start n8n

---

## Step 3 — Create credentials in n8n UI (manual, once)

Open `https://n8n.yourdomain.com` → Settings → Credentials → New

| Credential name | Type | Values |
|---|---|---|
| `n8n-claw Bot` | Telegram API | `TELEGRAM_BOT_TOKEN` from `.env` |
| `Postgres (Supabase)` | PostgreSQL | host: `127.0.0.1`, port: `5432`, db: `postgres`, user: `postgres`, password: `POSTGRES_PASSWORD` from `.env` |
| `Anthropic account` | Anthropic API | `ANTHROPIC_API_KEY` from `.env` |
| `Kong (Supabase)` | HTTP Header Auth | header: `apikey`, value: `SUPABASE_ANON_KEY` from `.env` |

> ⚠️ Use exactly these names — the scripts look them up by name/type.

---

## Step 4 — Run setup scripts

```bash
# Import workflows into n8n SQLite + insert API key
python3 scripts/setup-db.py --env-file .env

# Wire the credentials you just created into all workflows
python3 scripts/phase2-wire-credentials.py --env-file .env
```

---

## Step 5 — Final n8n UI step

In n8n UI, find the **n8n-claw Agent** workflow and toggle it **OFF** then **ON**.

This re-registers the Telegram webhook with a fresh secret token. Without this step, Telegram will return 403 on webhook calls.

---

## That's it

Send a message to your Telegram bot — it should respond.

## What each file does (reference)

| File | Purpose | Edit it? |
|---|---|---|
| `.env` | **All configuration lives here** | ✅ Yes — only this |
| `setup.sh` | Full automated installer | ❌ No |
| `scripts/generate-secrets.py` | Generates random Supabase secrets | ❌ No |
| `scripts/generate-n8n-api-key.py` | Derives n8n public-api JWT from encryption key | ❌ No |
| `scripts/setup-db.py` | Imports workflows, inserts API key, fixes cross-refs | ❌ No |
| `scripts/phase2-wire-credentials.py` | Wires n8n credential IDs into workflow nodes | ❌ No |
| `supabase/kong.yml` | Kong config template (placeholders) | ❌ No |
| `supabase/kong.deployed.yml` | Generated by setup.sh — never commit this | ❌ No |
| `supabase/migrations/001_schema.sql` | DB schema | ❌ No |
| `supabase/migrations/002_seed_en.sql` | Seed data template (USER_* vars) | ❌ No |
| `workflows/*.json` | Parameterized workflow templates | ❌ No |
| `docker-compose.supabase.yml` | Supabase Docker stack | ❌ No |
| `docker-compose.n8n.yml` | n8n Docker stack (docker mode only) | ❌ No |
| `n8n-systemd/n8n.service` | systemd unit (native mode) | ❌ No |
